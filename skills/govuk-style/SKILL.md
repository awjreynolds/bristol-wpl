# GOV.UK style

Use this skill for public, officer and cabinet prose. It is adapted for this Bristol WPL simulation from fofr's GOV.UK style skill and public GOV.UK content and publishing guidance.

Using this guidance does not make repo content official GOV.UK content or certified GOV.UK/GDS-compliant.

Sources:

- `https://gist.github.com/fofr/505e225f9bf5e839d30c12ba6bfa0be2`
- `https://guidance.publishing.service.gov.uk/writing-to-gov-uk-standards/writing-guidelines/`
- `https://guidance.publishing.service.gov.uk/writing-to-gov-uk-standards/style-guides/a-to-z-style-guide/`

## Core rule

Open the content up so a reader understands it first time. Do not dumb it down.

Do not simplify away legal, statutory, financial, evidence or no-go meaning.

## Apply it like this

1. Start with the reader need.
2. Put the answer first.
3. Use short sentences.
4. Use active voice.
5. Use concrete names, dates, IDs and file paths.
6. Split one long idea into separate sentences or bullets.
7. Replace vague words with direct words.
8. Use sentence case for new headings unless a controlled document title requires otherwise.
9. Use descriptive links and paths.
10. End with the next action or the blocker.

## Repository safeguards

Keep these exact controls:

- statutory and legal terms where precision matters
- source IDs, issue IDs, risk IDs, claim IDs and file paths
- status formulas for Stage 32A, Stage 33A and later stages
- no-go wording about approval, officer advice, procurement authority, consultation readiness, statutory readiness, professional assurance and WPL readiness
- evidence labels such as `Cited`, `Inference`, `Proposal`, `Assumption` and `Gap`

Do not rewrite legislation, direct quotes, CSV rows, register IDs, hash values, commands or raw source extracts for style.

## Avoid in public and officer prose

- vague words such as robust, key, landscape, ecosystem, going forward or deep dive
- passive voice when the actor is known
- long paragraphs
- unexplained acronyms
- emphasis with bold or italics
- "click here" or "read more"
- Latin abbreviations such as eg, ie or etc

## Before finishing

Check:

- the most important point is first
- a non-specialist can follow the text
- every sentence preserves the controlled meaning
- no no-go or evidence caveat was removed
- dates, source IDs, issue IDs and file paths still match the registers
