# Stage 0 and Stage 1 Foundation

Status: completed with conditions.  
Latest relevant commits: `d2a46b7`, `9b4416d`, `2dd72da`, `6884605`.

## Purpose

Stage 0 created the blank-repo architecture for a government-grade Bristol Workplace Parking Levy simulation. Stage 1 seeded and acquired the initial evidence base, extraction logs, source register and first simulated assurance trail.

## Key Data Points

- `evidence/source_register.csv` currently contains 111 source rows.
- 94 sources are downloaded and extracted.
- 16 sources remain seeded but not downloaded.
- 1 source acquisition failed: `SRC-ACADEMIC-0001`.
- Raw third-party PDFs are allowed only under `evidence/raw/**`.
- Officer review/distribution outputs must be editable Markdown, DOCX, XLSX, HTML or controlled data files, not authored PDFs.

## Main Findings

- The repo has a working source register, extraction manifest, risk register, approvals register, decision log, issues register and simulation sign-off register.
- Stage 1 created a credible evidence spine across Bristol governance, WECA/MCA context, DfT/HMT guidance, legislation, Nottingham precedent and media/context sources.
- Early appraisal and spatial reviews found the programme architecture credible but not decision-ready.
- Bristol public decision material supports continued development work, not implementation, preferred boundary, statutory consultation, FBC approval or statutory submission.

## What Stage 0/1 Did Not Claim

- It did not create an OBC or FBC.
- It did not select a scheme boundary.
- It did not create model outputs.
- It did not certify statutory compliance.
- It did not produce officer-distribution PDFs.

## Continuing Issues

- Source notes are still incomplete for acquired priority sources.
- `SRC-ACADEMIC-0001` remains unavailable and cannot support independent Nottingham congestion claims.
- Individual TAG units, TAG databook, VFM framework and appraisal source notes remain incomplete.
